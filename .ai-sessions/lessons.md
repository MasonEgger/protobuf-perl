# Lessons Learned

## Recent

- The native `.proto` parser can't tell enum from message syntactically, so `Grammar::_parse_field_type` tags BOTH `type => 'message'`; `Schema::_resolve_message` must correct it from the resolved ref's class (`$ref->isa('Protobuf::Schema::Enum') ? 'enum' : 'message'`) right after `set_type_ref`, or enum fields keep `is_message` true and the codec sends the enum integer down the embedded-message path тЖТ dies "unknown message type". Invisible to conformance because the DescriptorSet/protoc path already sets `type => 'enum'`; only the native-parse тЖТ codec path is affected. Needs a `Field::set_type` companion to `set_type_ref` (2026-06-06)
- When Test::Pod::Coverage isn't installed, audit POD coverage with a dependency-free grep: compare each module's public `^\s*(method|sub)\s+[a-z]\w*` names (drop leading-underscore ones) against its `=item`/`=head2` POD identifiers. Pod::Coverage matches a public sub only by a NAMED `=item C<foo>`/`=head2 foo` тАФ an inline `C<foo>` mention or a SYNOPSIS code block does NOT count, so accessor-heavy `class` modules need an explicit `=head1 ACCESSORS` list of `=item`s (2026-06-01)
- Keep the `cpanfile` test-prereq aligned with the framework the tests actually `use`: this project declared `Test2::V0` but every `t/*.t` uses core `Test::More` (and Test2 isn't even installed), so `dzil`/`cpanm --installdeps` would pull a phantom dep. Author-only `xt/` tools (Test::Pod, Test::Pod::Coverage) belong in an `on 'develop'` block, not `test` (2026-06-01)
- Guard author tests so a missing module SKIPS, never fails: `eval { require Test::Pod; Test::Pod->import; 1 } or plan skip_all => '...';`. Put them under `xt/` (not `t/`) so the `prove -lr t` gate never runs them, and gate integration smokes (`t/integration/*.t`) on an env-var path with `plan skip_all` so the suite stays green without the large external graph (2026-06-01)
- To keep an AOT code generator from drifting from the runtime class builder, make each generated `.pm` reconstruct its `Schema::File` as STATIC Perl (a `Schema::Serializer` rendering File/Message/Field/Oneof/Enum constructors тАФ drop `type_ref` and call `$schema->resolve` at load time instead) and then install every class through the same `Proto3::Class::Generator->build`. The AOT module reuses the runtime build path verbatim, so round-trip/oneof/map/nested behavior is shared by construction and the generated module needs no parser/descriptor-set dependency (only the grep-guarded schema+codec+WKT layers) (2026-06-01)
- For runtime-generated proto classes, install accessors as closures into the target package's symbol table (`no strict 'refs'; *{"${pkg}::$m"} = sub {...}`) rather than `feature 'class'` + string eval тАФ closures capture the descriptor/field metadata directly, sidestep the feature-'class' package-scoping traps inside generated blocks, and avoid the eval-compiles-but-fails-at-runtime risk. Key the instance hash by proto field NAME (not the possibly keyword-mangled accessor name) so `to_hashref`/codec always see wire names (2026-06-01)
- Under `feature 'class'`, a bareword sub imported at file scope (`use Cwd qw(abs_path)`) is INVISIBLE inside class methods тАФ the unqualified `abs_path(...)` resolves to `<ClassPkg>::abs_path` and dies "Undefined subroutine" at runtime (compiles fine with `-c`). Fully-qualify (`Cwd::abs_path`) or assign to a pre-class `my` lexical. Same trap as constants, now confirmed for imported subs (2026-06-01)
- A Grammar helper returning a list (e.g. `_parse_field_type` -> `($type, $type_name)`) must be called with a parenthesized lvalue (`my ($t) = ...`); a bare `my $t = ...` takes the LAST list element (the undef type_name), silently dropping the value. Bit the map-key parse until switched to list context (2026-06-01)
- Perl 5.38.2 mis-parses a file-scope `sub (signature)` тАФ named (`my sub f ($x){}`) OR anonymous (`my $f = sub ($x){}`) тАФ when a `class` block follows it, dying "Subroutine attributes must come before the signature". Wrap such helper coderefs in a `do { ...; ($a,$b) }` block (the trick `%SCALAR_TYPE` already uses) to insulate the signatures (2026-05-31)
- proto3 encodes a negative `int32`/`int64` as its full 64-bit two's complement (`2**64 + value`, always a 10-byte varint) тАФ NOT as a plain varint (which rejects negatives) and NOT as zigzag (that's only `sint32`/`sint64`). Decode reverses it: a varint with bit 63 set is `value - 2**64` (2026-05-31)
- `Proto3::Wire::decode_zigzag32/64` return ONLY the value, not `($value, $rest)` like the other `decode_*` тАФ a codec consumer must read the varint separately to recover `$rest`, or the decode loop ends early and every field after a sint silently defaults (2026-05-31)

## Architecture
- `Schema->resolve` must walk `$schema->files` (recursing into each message's nested_messages), NOT the flat `all_messages` index, because a field's resolution scope needs its declaring file's `package` as current_package and its owning message's `full_name` as current_message тАФ the flat index has no back-pointer to the owning file/package (2026-05-31)
- The native parser tags every named-type field `type => 'message'` (enum vs message is undecidable syntactically); `Schema::_resolve_message` corrects it from the resolved ref's class via a narrow `Field::set_type`. Conformance only exercises the DescriptorSet path (correct type from the start), so native-parse-only type bugs hide from it тАФ add a parser тЖТ codec round-trip test for such cases (2026-06-06)

## Testing
- A protoc differential oracle catches wire bugs hand-rolled fixtures miss: earlier codec round-trip tests were green despite the negative-int and zigzag-`$rest` bugs because they only used positive ints and tested zigzag as the last field. Build the oracle test BOTH directions (our-encodeтЖТ`protoc --decode` and `protoc --encode`тЖТour-decode) (2026-05-31)

## Codegen

## Perl
- Under `feature 'class'`, an imported bareword sub (`use Cwd qw(abs_path)`) lands in the file package and is invisible inside class methods (`abs_path(...)` resolves to `<ClassPkg>::abs_path`, dying at runtime though `-c` passes); fully-qualify (`Cwd::abs_path`) or use a pre-class `my` lexical (2026-06-01)
- A class helper returning a list must be called in list context (`my ($x) = ...`); a scalar assignment takes the last list element, silently swallowing earlier values (2026-06-01)
- Perl 5.38.2 mis-parses a file-scope `sub (signature)` (named or anonymous) immediately before a `class` block ("Subroutine attributes must come before the signature"); wrap the coderefs in a `do {}` block to insulate the signatures (2026-05-31)
- `decode_zigzag32/64` return a single value (not `($value, $rest)`), unlike every other `Proto3::Wire` `decode_*`; a consumer needing the remainder must decode the varint separately for `$rest` (2026-05-31)
- A typed exception thrown via `Class->throw` must have its `:isa` declaration committed alongside the caller; an undeclared package autovivifies but `->throw`/`isa` fail at runtime, so the bug only surfaces on a clean checkout (2026-05-31)
- This box's Perl 5.38.2 takes `:param` but rejects `:reader` on `field`; declare readers as explicit `method foo { $foo }` (2026-05-30)
- Enable new-OO syntax with `use feature 'class'; no warnings 'experimental::class';` (not `use experimental 'class'`, whose module is absent) (2026-05-30)
- For a dual class/instance method (e.g. `throw`) under the `class` feature, use a plain `sub` inside the class block тАФ `die ref $invocant ? $invocant : $invocant->new(@args)` тАФ which inherits to subclasses cleanly (2026-05-30)
- Spec ┬з4.10 wants the whole exception hierarchy in one file (multiple `class` blocks); subclasses are bare `:isa(...)` with zero code тАФ overload, `throw`, and readers live only on the base and inherit (2026-05-30)

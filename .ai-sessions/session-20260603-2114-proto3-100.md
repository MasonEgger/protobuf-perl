## Fixed in this batch (two parallel streams, orchestrator-verified + committed)

### Codec layer (lib/Proto3/Codec.pm, lib/Proto3/Conformance.pm)
- ENUM stored as a signed 32-bit int (over-width varint wraps to 32 bits), like int32.
- Message merge: multiple occurrences of the same singular message field MERGE
  recursively at the wire-byte level (protoc semantics), instead of the later
  occurrence replacing the earlier and dropping sub-fields.
- IllegalZeroFieldNum: decode rejects a field number of 0 (was silently accepted).
- UnknownVarint: the conformance testee now builds its codec with
  preserve_unknown_fields => 1 so unknown fields round-trip protobuf->protobuf.
- Tests: t/codec/{enum_width,message_merge,zero_field_num,conformance_preserves_unknown}.t

### WKT/JSON layer (WKT/Timestamp.pm, Duration.pm, Wrappers.pm, JSON.pm)
- Timestamp JSON parses numeric zone offsets (+HH:MM / -HH:MM) and normalizes to UTC.
- Timestamp/Duration range bounds enforced on BOTH JSON input and protobuf-input
  -> JSON-output (reject out-of-range -> Proto3::Exception::JSON::WKT).
- Wrapper types (BoolValue/BytesValue/...) coerce their JSON form to/from the codec
  representation (JSON bool <-> 1/0; base64 <-> raw bytes).
- BoolMapField: JSON object keys "true"/"false" coerce to bool map keys.
- Tests: t/wkt/wkt_json_conformance.t
